# 2주차 과제
# 1. HTTP 내용 정리

## HTTP (HyperText Transfer Protocol)
서버와 클라이언트가 서로 데이터를 주고 받기 위해 사용되는 통신 규약으로 정의된 규칙에 따라 문서를 전송하기 위해 사용합니다.


## HTTP Message
HTTP Message는 서버와 클라이언트 간에 파일 등을 주고 받는 통신의 기본 단위로 데이터를 교환하기 위해서 사용되며 요청 메시지와 응답 메시지가 있습니다.

## HTTP 요청 메소드
HTTP 요청 메소드는 웹 서버에게 어떤 작업을 수행하길 원하는지 알리는 명령으로 구체적인 작업을 명확하게 지시하기 위해서 사용됩니다. 

대표적인 메소드로는 GET(읽기), POST(생성), DELETE(삭제), PUT(리소스 전체 업데이트), PATCH(리소스 부분을 업데이트)가 있습니다.

## HTTP 상태 코드
HTTP 상태 코드는 3자리의 숫자로 요청의 처리 상태를 나타내는 코드입니다. 

100번대는 정보, 200번대는 성공, 300번대는 리디렉션, 400번대는 클라이언트 오류, 500번대는 서버 오류를 의미합니다. 

## HTTP URL
HTTP URL은 주소를 의미하며 리소스의 위치와 종류를 나타내기 때문에 웹 상의 리소스를 식별하고 접근하기 위해서 사용합니다. URL은 스킴, 도메인, 포트, 리소스 경로, 매개변수로 구성되어있습니다.

## HTTP 경로 변수와 쿼리 스트링
HTTP 경로 변수는 경로 부분에 포함된 특정 자원을 식별하는 변수로 특정 자원에 접근하기 위해 사용되며 리소스, 식별자 등을 URL에 포함시켜 표현합니다.

HTTP 쿼리 스트링은 일반적으로 URL 끝에 "?" 기호를 붙이고 그 뒤에 키와 값 형태로 데이터를 전달합니다. 이는 웹 페이지에 대한 추가 정보를 서버에 전달하기 위해 사용됩니다.

## JSON
JSON은 구조화된 데이터를 표현하기 위한 문법을 의미하며 가볍고 구조화된 데이터 형식으로 데이터를 주고 받기 위해 사용됩니다. 대체로 Dictionary 형태나 배열로 사용됩니다.

## REST API
REST API는 REST 아키텍처 스타일을 따르는 응용 프로그램 인터페이스를 의미합니다. REST(Representational State Transfer)란 자원을 이름으로 구분하여 해당 자원의 상태를 주고받는 것을 의미하는데 HTTP URI를 통해 자원을 명시하고, HTTP Method를 통해 해당 자원에 대한 CRUD 오퍼레이션을 적용하는 것을 의미합니다.

이를 사용하는 이유는 일관된 통신 규칙을 마련하여 업무 효율을 높이고, 단순한 URL 구조를 통해 요청의 목적을 쉽게 이해할 수 있게 하기 위해서입니다.
